# WriteBuddy Documentation

이 폴더는 WriteBuddy 프로젝트의 모든 문서를 포함합니다.

## 폴더 구조

```
docs/
├── README.md                 # 이 파일 - 문서 구조 개요
├── design/                   # UI/UX 디자인 및 디자인 시스템
│   ├── design-summary.md     # 디자인 요약
│   ├── week1-design-spec.md  # Week 1 디자인 명세
│   └── week1-frontend-plan.md # Week 1 프론트엔드 계획
├── development/              # 개발 가이드 및 구현 문서
│   └── DEVELOPMENT.md        # 개발 가이드
├── team/                     # 팀 구조 및 협업 관련
│   ├── team-structure.md     # 팀 구조
│   └── collaboration-test-result.md # 협업 테스트 결과
└── project/                  # 프로젝트 전체 관리 문서
    └── writebuddy-roadmap.md # WriteBuddy 로드맵
```

## 주요 문서 설명

### 프로젝트 개요
- **README.md**: 프로젝트 소개, 기술 스택, API 명세 등

### 디자인
- **design-summary.md**: 디자인 철학 및 UI 원칙
- **week1-design-spec.md**: Week 1 기능 디자인 명세
- **week1-frontend-plan.md**: Week 1 프론트엔드 구현 계획

### 개발
- **DEVELOPMENT.md**: 개발 환경 설정 및 가이드

### 팀
- **team-structure.md**: 5인 팀 구조 및 역할 분담
- **collaboration-test-result.md**: 협업 테스트 결과 및 피드백

### 프로젝트 관리
- **writebuddy-roadmap.md**: 기능 우선순위 및 4주 로드맵

## 문서 작성 규칙

1. **파일명**: 영어로 작성, 하이픈(-) 사용
2. **내용**: 한국어 우선, 코드 예제는 영어
3. **형식**: Markdown 사용
4. **버전**: 날짜 기반 버전 관리

## 기여 방법

새 문서 작성 시:
1. 적절한 카테고리 폴더 선택
2. 파일명 규칙 준수
3. 이 README.md에 링크 추가
4. 팀 리뷰 후 승인
